# Documentation

This directory contains hand-written guides for software developers, data scientists,
and data engineers who use or extend `spark-preprocessor`.

## Guides

- `overview.md`: What the library does and the end-to-end flow.
- `architecture.md`: Compiler/runtime architecture and artifact layout.
- `pipeline.md`: Pipeline YAML reference.
- `mapping.md`: Mapping YAML reference.
- `profiling.md`: Profiling notebook configuration and behavior.
- `features.md`: Feature authoring guide (registry, params, assets).
- `runtime.md`: Databricks runtime execution model.
- `testing.md`: Local testing strategy and expectations.
- `troubleshooting.md`: Common errors and fixes.
- `api-reference.md`: Curated public API and CLI entrypoints.
